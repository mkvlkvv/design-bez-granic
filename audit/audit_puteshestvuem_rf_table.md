**Источники требований:** WCAG 2.1 AA, Постановление Правительства РФ № 102 от 07.02.2026, ARIA APG.

**Метод:** разбор HTML-ответа страниц (структура, семантика, ARIA, alt, формы, порядок чтения). Визуальные проверки контраста и фокуса требуют запуска в Chrome — отмечены отдельно.

## Что требует ручной проверки в Chrome (недоступно из HTML-дампа)

1. **Контраст** текста на цветных кнопках-тегах, на полупрозрачных баннерах над фото, серого плейсхолдер-текста → axe DevTools / WAVE
2. **Индикатор фокуса** при Tab по пагинации, фильтрам, тегам, кнопкам карусели → визуально
3. **Поведение модалки поиска и бургер-меню**: ловушка фокуса внутри, Esc закрывает, фокус возвращается на триггер
4. **VoiceOver-прохождение** статьи целиком: посчитать, сколько лишних «картинка», «ссылка», «группа» объявляется
5. **Клавиатурный обход**: есть ли места, где Tab «залипает» (особенно карусели, Яндекс-карта)
6. **Zoom 200% / 400%**: перекрывается ли контент, появляется ли горизонтальный скролл (сейчас заблокирован `maximum-scale`, но для ПК это не работает)
